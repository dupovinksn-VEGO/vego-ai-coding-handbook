Logistics Intelligence

物流與供應鏈智庫地圖

VEGO 寶典會逐步沉澱物流、供應鏈、關務、倉儲、運輸、客服、報價與財務流程。 每個主題都可以連到文章、系統來源、Sandbox Skill，最後變成新項目可套用的 SOP。

Inbound

Pre-alert / PL / Devanning

從客戶文件、PL/CI、container、拆櫃到上架前狀態，重點是 parser 成功後 downstream handoff 也要可操作。

Warehouse

Inventory / SKU / Allocation

區分 planning inventory 與可出貨庫存，避免 PREVIEW 狀態被錯誤分配到 outbound。

Outbound

Dispatch / BOL / POD

追蹤派送、承運商交接、BOL、POD、異常回報與客戶通知。

Customs

HTS / Broker Review / PL-CI

維持 regular cargo 與 express/small-parcel 雙軌，並保持 broker-facing PL/CI 正常可用。

Customer

SOP / Exception / Draft

把客戶 SOP、異常類型、回覆草稿、升級規則整理成可審核的客服知識。

Finance

Quote / Invoice / Billing Gate

報價規則、費率查詢、invoice send gate 與異常費用要能追蹤、驗證、回溯。

AI 自動化到物流 AGI 的領域分層

每一層都先進 Sandbox Review，再決定是否進入正式寶典與可執行 Skill。

Knowledge

知識層

文章、電子書、SOP、客戶案例、系統截圖與錯誤案例先沉澱為可搜尋的智庫條目。

Automation

AI 自動化層

把重複流程拆成可審核任務：文件抽取、狀態比對、異常分類、報表生成、回覆草稿。

Agents

智能體層

為 Inbound、Warehouse、Customs、Dispatch、Customer、Finance 設計專屬 Agent 職責與交接邊界。

Robotics

機器人協作層

把掃碼、盤點、揀貨、AMR、視覺檢查與 WMS 狀態串接成「可驗收、可回退」的現場任務。

Control Tower

物流 AGI 控制塔層

讓多個 Agent 共同做規劃、監控、模擬與異常升級，但關務、付款、對客承諾仍保留人工批准閘。

Safety

人審與合規層

所有外部 Skill、客戶訊息、關務輸出、刪除動作與生產系統寫入，必須先通過 Sandbox 與責任人確認。

Intelligence Hub

你可以登記外部電子書、訂閱文章、GitHub repo、Netlify/Vercel 系統、內部工具網址。 系統先建立來源卡與吸收計畫，再決定是否整理成寶典內容或 Sandbox Skill。

吸收邊界

外部電子書 / 訂閱內容：可以保存連結、個人摘要、短摘錄與方法論改寫；不做整本搬運，不繞過 DRM/paywall。

你自己的系統：可以登記 URL 與目標，先用 read-only 方式抽取可學習的流程、頁面模式、錯誤處理與 Skill 線索。

外部 repo：先只讀 README、license、SKILL.md、package metadata，不自動執行命令。

Sandbox Review

先看效果，再決定是否進正式寶典

每個新增來源會產生收藏卡、風險卡、轉換建議與 VEGO 適用度。

1

提交 URL / 文章

貼文章、GitHub repo 或安裝命令，系統抽取 Skill 名稱、用途與來源。

2

只讀典藏

只讀抓取 README、SKILL.md、package 資訊、release 與 license。

3

安全審核

標記 shell、網路請求、token、刪檔、外部 API、授權限制等風險。

4

VEGO 化轉換

轉成繁中主體、英文摘要、適用場景、測試案例與使用限制。

目前策略： 外部 Skill 先放入 Sandbox，不直接自動啟用。你確認後才升級到正式 Skill Vault。

每個外部來源都會留下狀態，方便你決定要收藏、轉換、批准或暫停。
