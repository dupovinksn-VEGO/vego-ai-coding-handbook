Sandbox Review

先讓你看效果，再決定是否進正式寶典

每個新增來源會產生「收藏卡、風險卡、轉換建議、VEGO 適用度」。

1

提交 URL / 文章

你貼文章、GitHub repo 或安裝命令，系統抽取 Skill 名稱、用途與來源。

2

自動下載典藏

只讀抓取 README、SKILL.md、package 資訊、release 與 license。

3

安全與適用度審核

標記 shell、網路請求、token、刪檔、外部 API、授權限制等風險。

4

VEGO 化轉換

轉成繁中主體、英文摘要、適用場景、測試案例與使用限制。

目前策略： 外部 Skill 先放入 Sandbox，不直接自動啟用。你確認後才升級到正式 Skill Vault。

審核駕駛艙

每個外部來源都會留下狀態，方便你決定要收藏、轉換、批准或暫停。

Weekly Update

每週自動升級，但正式更新前保留人工確認

AI 工具變化很快，每週檢查官方 changelog、已收藏 repo、你新增的文章來源。

每週檢查來源

  • OpenAI / Codex docs 與 changelog
  • GitHub Skill repositories release / commits
  • Vercel、Netlify、Cloudflare AI tooling
  • 你不定時補充的文章與網頁

輸出內容

  • weekly-update-report.md
  • pending skill cards
  • 安全風險分級
  • Netlify preview / sandbox page

批准後更新

  • 正式寶典章節
  • 正式 VEGO Skill
  • 外部 Skill 典藏紀錄
  • CHANGELOG
