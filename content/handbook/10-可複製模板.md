# 可複製模板

把這些模板貼進 Codex 或新項目文件中使用。

## PR Review Prompt

```text
請以物流系統與客戶體驗角度 review 這次改動。
優先找 bug、資料風險、權限漏洞、未驗證流程與客戶可見錯誤。
請列出文件/行號、風險等級、建議修法與需要補的測試。
```

## Architecture Prompt

```text
請先不要寫程式。請根據以下物流流程設計資料模型、狀態流、API、權限、錯誤處理與驗收測試。
Done when: 產出可由 Agent 實作的分階段任務清單。
```

## Tool Comparison Prompt

```text
請比較以下 AI Coding 工具或 Skill：
1. 是否能讀整個 repo？
2. 是否能規劃、實作、測試？
3. 是否會要求過高權限？
4. 是否能生成可 review 的 diff？
5. 是否適合 VEGO logistics workflow？
```

## 完成前檢查 Prompt

```text
請在宣稱完成前重新跑驗證：
- check/build/test 指令
- 主要頁面 smoke test
- 手機版布局
- 空資料與錯誤狀態
- 最終列出改動、驗證結果、剩餘風險
```

## English Summary

Copy-ready templates help VEGO start new projects with clear review and architecture standards.
